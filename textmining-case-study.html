20th Century Text Mining & Network Analysis

Python • Selenium • NLTK • TextBlob • spaCy • NER • Network graphs • Web scraping

Project Overview

This end-to-end NLP project scrapes a Wikipedia article on 20th century key events, performs text mining and linguistic analysis, then uses Named Entity Recognition (NER) to extract country co-occurrences and build an interactive network graph showing which nations appeared together most frequently in historical events.

The Pipeline

The project covers the full data science pipeline from raw web data to interactive visualization — scraping, cleaning, linguistic analysis, entity extraction, relationship modeling, and network graph construction.

Phase 1 — Web Scraping with Selenium

Used Selenium with ChromeDriver to scrape the full text of the Wikipedia "20th century" article programmatically. The raw scraped text required cleaning to remove navigation elements, UI chrome, and metadata that Selenium captured alongside the article content.

# Scrape Wikipedia page text using Selenium
from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Chrome()
driver.get("https://en.wikipedia.org/wiki/20th_century")
text = driver.find_element(By.ID, "mw-content-text").text
driver.quit()

# Save to file for downstream analysis
with open("20th_century_key_events.txt", "w") as f:
    f.write(text)

Phase 2 — Text Mining with NLTK & TextBlob

After cleaning, the text was tokenized and analyzed using NLTK and TextBlob:

  • Word frequency analysis — before and after stopword removal revealed that "war", "world", and "soviet" dominated historical content
  • Part-of-speech (POS) tagging using TextBlob to identify nouns, verbs, and named entities
  • Top 15 nouns extraction to surface the most historically significant themes
  • Stopword removal demonstrated how cleaning transforms interpretability — raw frequency was dominated by common English words

Phase 3 — Named Entity Recognition (NER) with spaCy

Used spaCy's en_core_web_sm model to extract geopolitical entities (GPE tags) from every sentence. Entity aliases were normalized — "UK", "U.K.", and "Britain" all mapped to "United Kingdom" — to ensure consistent relationship counting.

import spacy

nlp = spacy.load("en_core_web_sm")
doc = nlp(clean_text)

# Extract sentences containing multiple country mentions
sentence_entities = []
for sent in doc.sents:
    entities = [ent.text for ent in sent.ents if ent.label_ == "GPE"]
    if len(entities) > 1:
        sentence_entities.append(entities)

Phase 4 — Country Co-occurrence Network

Countries appearing together in the same sentence were treated as connected nodes. The number of co-occurrences became the edge weight. This produced a graph where heavily connected nations reflect genuine historical entanglement — alliances, conflicts, and shared events in the 20th century record.

Key Findings

  • After stopword removal, the most frequent words were "war", "world", "soviet", "united", and "states" — reflecting the century's dominant historical themes
  • The United States, Soviet Union, Germany, and United Kingdom formed the most densely connected cluster in the co-occurrence network
  • Country co-occurrence patterns closely mirror known geopolitical relationships — Cold War rivals (US/USSR) appeared together most frequently
  • Alias normalization was critical — without it, "US", "U.S.", and "United States" would appear as three separate unconnected nodes

Technical Decisions

The choice to use sentence-level co-occurrence (rather than document-level) was deliberate: it captures meaningful proximity between countries rather than just their presence in the same article. Two countries mentioned in the same sentence about a specific event share a much more concrete relationship than two countries simply both mentioned across a long document.

Tools & Libraries

  • Selenium — automated browser scraping
  • NLTK — tokenization, stopword removal, frequency analysis
  • TextBlob — POS tagging, noun/verb extraction
  • spaCy — Named Entity Recognition (NER)
  • NetworkX + pyvis — network graph construction and interactive HTML export
  • Pandas — data structuring and CSV export of edge weights

Conclusion

This project demonstrates the full NLP pipeline from raw web data to insight. The network visualization makes an abstract question — "which countries were most historically connected in the 20th century?" — immediately and intuitively answerable through graph structure. It also highlights the importance of careful text cleaning and entity normalization as prerequisites for meaningful analysis.
